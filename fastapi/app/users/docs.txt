1. get_password_hash -> принимает пароль, хеширует его и возвращает
2. verify_password -> принимает два пароля (первый в обычном виде, 
второй хешированный), сравнивает их и возвращает boolean
3. create_access_token -> принимает некий словарик с данными о пользователе 
(в нашем случае с id, защита которого не в приоритете), дополняет его отдельным
аргументом exp (показывающим длительность этого токена), кодирует его с 
использованием ключа и возвращает в виде токена в защифрованном виде 
4. authenticate_user -> получает данные email и password, возвращает user если
таковой существует и его пароль равен передаваемому
