1. dao -> наследуясь от .dao работает с
async_maker_session для работы с бд

2. models -> создает класс модели для бд, наследуясь
от Base(DeclarativeBase)

3. router -> использует локальный dao для работы с бд
и реализует работу с роутами (путями), использует
schemas для валидации

4. schemas -> содержит pydantic модели для валидации 
данных в router
